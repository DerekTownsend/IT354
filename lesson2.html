Functions

Named Functions

Name functions are functions that that have names. They are declared using the function declaration followed by the name of the function. It is the most basic way to create a function.

function add(num1, num2){
return num1 + num2;
}
console.log(add(1,2)) // 3

Function Expressions

These are used to help integrate the usefulness of let and const. Let and const helped stop the problems that hoisting can cause and if you wanted a named function it would get hoisted. Function expressions stop hoisting and makes your code an bit more modularized.

const subtract = function(num1, num2){
return num1 - num2;
}
console.log(subtract(1,2)) // returns -1

Arrow Functions

One of the newer addtions to JS and it is very similar to function expressions except that it allows you to condense your code down and maintain your scope of this. We haven't really covered what this is but I will just stay away from it because it is very confusion to explain. It helps you condense your code because you can you can get rid of the parentheses if you have one variable and also remove your return statement and curly braces if it is only one line/ your return statement is your only line of code.

const divide = (num1, num2) => {
return num1 / num2;
}
console.log(divide(1,2)) // returns 0.5
// can also be written
const square = num1 => {
return num1 ** 2
}
console.log(square(2)) // 4
// or
const cube = num1 => num1 ** 3
console.log(cube(2)) // 8

Other Topics

Optional Arguments

Javascript ignores everything extra that you put in your function call. If you only accept one parameter in a function, but when you make the call you accidentially put in two it will only pay attention to the first.

function printName(firstName){
console.log(`Hello, ${firstName}!`)
}
printName("Dave",5,"forty") // "Hello, Dave!" Ignores Additional parameters and doesn't throw error
printName() // "Hello, undefined!" because it requires a parameter otherwise undefined

Rest Parameters

Essentially this lets you take in as many parameters as you want and puts them into an array to be processed. Good for batch processing/doing a very repeatable and reliable action over and over again.

function printNames(...names){
// takes all of the parameters entered and puts them into an array
for (name of names){
console.log(`Hello, ${name}!`)
}
}
printNames("Dave","Jane", "John","Satania")
// prints:
// Hello, Dave!
// Hello, Jane!
// Hello, John!
// Hello, Satania!
// should be used at the end otherwise it takes assumes all parameters are part of the spread operator

Scope

Scope is what your function/code has access to. What can variables can your function access at any time. Variables defined within functions can't be accessed out side the fuctions(without returning them or other methods), but variables declared outside of the function can be accessed by the function(most of the time).

// Ex 1

function printHeight(height){
console.log(`You are ${height}cm tall`)
}
printHeight(1) // "You are 1cm tall"
