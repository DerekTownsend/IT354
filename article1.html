A Brief History of ECMAScript

So from my basic understanding of ECMAscript is that it is the standards and common practices for Javascript. ECMAscript essentially dictates the way that that javascript should be written, what built in functions currently exist, and what features might be added in the future. An important change that was made was the inclusion of let, const, and arrow function which I talk about in lessons 1 and 2 of this site. This inclusion along with other changes made at that time changed the way that we wrote javascript and helped improve a lot of frameworks.

There have been 10 versions of ECMAscript release since JavaScript's creation in 1995. The most current version being the 2019 version. The big changes I mentioned earlier were mentioned in the ES6 version. One of the changes I excluded were class declarations making the language far more viable for object oriented programming. JavaScript did have ways of doing it previously but they were a bit more tedious and visulally unappealing. It still revolves around protypical inheritance, but it is more hidden and less obvious. The organization that dictates ECMAScript standards are TC39 which is made up of industry giants that the language's current state affects most and standardized by ECMA International.
