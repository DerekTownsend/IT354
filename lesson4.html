Prototypal Inheritance

The back bone of JS classes and objects. Allows for us to create functions within classes that still maintain the classes scope of "this". Also allows for us to build out more functionality for other classes. Helped make object oriented programming more viable and widely accepted.

This

A JS keyword that allows for you to refer to a specific object/window/node's attributes, values, and functions. Very useful if you will to access certain variables or functions that are nested within a class.

Syntactic Sugar

Same code functionally but is written differently to make it easier for other devs to understand.


class Dog{
  constructor(name){
    this.name = name
  }
  bark(){
    console.log(`${this.name} barks`)
  }
}

function Cat(name){
  this.name = name
}
Cat.prototype.meow = function meow(){
    console.log(`${this.name} meows`)
}

let dog = new Dog("Daniel")
dog.bark() // "Daniel barks"
let cat = new Cat("Mavis")
cat.meow() // "Mavis meows"

Global Scope

The global scope refers to all things with in the JS node. If a variable is stored on the global scope it can be accessed by any JS files within the same scope. To store a variable on the global scope you simply set the variable equal to the object or value that you want to be tracked. Not reccommended to store variables here.


//Within Node
console.log(window === this) // false
a = "Global"
console.log(this.a) // "Global"

Window Scope

The window scope refers to everything within the browsers window. Not reccommended to store variables here.


//Within Node
window.a = "Window"
//In window
a = "Window"
console.log(window === this) //true
console.log(window.a) // "Window"
console.log(this.a) // "Window"
