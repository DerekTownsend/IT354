Lesson 3

Objects

Objects are what we use to store information about something specific with multiple characteristics that may fit under multiple data types. An object is surrounded in curly braces or "{ }" and has a key value pair that is structured {key:value}. Each key and value are seperated by commas. There are multiple ways to declare/create objects.

New Object Declaration and Dot Notation

The new object declarations is a very basic way of creating an empty object that you can then later add on to and manipulate. You can add key value pairs by using dot notation or bracket notation. With dot notation you take the name of the variable referencing the new object created and put a "." and the name of the key and set it equal to the value you want that key to refer to.


const airplane = new Object()

airplane.technicalName = "Fairchild Republic A-10 Thunderbolt II"
airplane.nickname = "A10 Warthog"
airplane.knownFor = "going BRRRRRRTTTTTTTTTTT"

airplane.showDetails = function(){
  console.log(`I am the ${this.technicalName} also known as ${this.nickname} and I am known for ${this.knownFor}.`)
}

airplane.showDetails()
 // I am the Fairchild Republic A-10 Thunderbolt II also known as A10 Warthog and I am known for going BRRRRRRTTTTTTTTTTT.

Bracket Notation

Bracket notation is similar to dot notation except that you can now call variables within the brackets and you can't declare functions using bracket notation. At least not the same way with dot notation.


const airplane = new Object()

airplane["technicalName"] = "Fairchild Republic A-10 Thunderbolt II"
airplane["nickname"] = "A10 Warthog"
airplane["knownFor"] = "going BRRRRRRTTTTTTTTTTT"

// doesn't work
 airplane["showDetails"] = function() {
   console.log(`I am the ${this.technicalName} also known as ${this.nickname} and I am known for ${this.knownFor}.`)
 }

// does work
 const temp = function(){
    console.log(`I am the ${this.technicalName} also known as ${this.nickname} and I am known for ${this.knownFor}.`)
}
airplane["showDetails"] = temp

airplane.showDetails()
// I am the Fairchild Republic A-10 Thunderbolt II also known as  A10 Warthog and I am known for going BRRRRRRTTTTTTTTTTT.

Object Literals

Object literals are you expliclty typing out your key and value pairs within the brackets. It is the most simple and common (in my opinion) way to create basic objects. In order to add functions to an object you use the "get" keyword before the function.


const userInfo = {
    firstName: "Micheal",
    lastName: "Alfred",
    get intro() {
    console.log(`My name is ${this.firstName} ${this.lastName}`)
  }
}

userInfo.intro
//My name is Micheal Alfred

console.log(userInfo.firstName,userInfo.lastName)
// Micheal Alfred

Classes

Classes are essentially JavaScript objects that have some more functionality and syntactic sugar. Syntactic sugar is something added that makes the code more readable for developers. 


class Rectangle {

 constructor(height, width){
   this.height = height
   this.width = width
 }

 findPerimeter(){
   return (2*this.height) + (2*this.width)
 }

 findArea(){
   return this.height * this.width
 }
}
const rectangle1 = new Rectangle(8,9,)

console.log(rectangle1.findPerimeter()) // 34
console.log(rectangle1.findArea()) // 72
